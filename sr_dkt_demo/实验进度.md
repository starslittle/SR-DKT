## 一、实验设计（不变的大前提）

- 数据集 **MOOCCubeX**；主实验设定：**10% 分层训练子集（96,044 学生）+ 全量 val/test 评估**。
- 对比**方案②**：标准基线在官方 **pyKT** 框架跑；**SR-DKT** 在自研框架跑；用 **DKT-Vec** 做跨框架对齐锚点
  （与 pyKT-DKT 在同一 10% 数据上比 AUC，差 `<0.005` 即证明两框架等价、可同表比较）。
- 自研框架真正要产出的基线只有 3 个：**DKT-Vec**（对齐）、**LBKT**、**BEKT**；
  标准基线 **DKT / DKVMN / SAKT / AKT / simpleKT / DKT-Forget** 全部走 pyKT（自研框架里那几个同名模型只作预实验，不进主表）。

## 六、避坑备忘

- `train.py` / `run_baselines.py` 支持 `--data-dir data/mooc_10pct`；**`ablation/ablation_study.py` 和 `evaluate.py` 不支持**，需把 `data/mooc` 软链接指向子集，跑完还原（见 AutoDL实验流程.md 5.5/5.6）。
- 子集目录里 val/test 是**绝对路径软链接**指向 `/root/autodl-tmp/sr_dkt/data/mooc/`；移动 data/mooc 会断链，靠 `ln -s` 在老地址立路标修复。
- Windows Git Bash **没有 rsync**；大文件传输用 `sftp` 的 `reput`（可断点续传），且 reput 要求远端文件已存在（先 `touch`）。
- pyKT 跑基线必须用 clone 的源码（pip 版缺 examples/configs）。
